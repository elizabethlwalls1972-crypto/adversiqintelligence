                    SECTION 2 — THE PROBLEM THIS SYSTEM SOLVES

================================================================================

Regional communities around the world are the backbone of their nations. They
produce the food, the energy, the raw materials, and the workforce that
national economies depend on. Yet they are systematically denied the tools to
compete for global investment and partnerships.

THE GAP:
--------
• A regional council in the Philippines, New Zealand, or Australia that wants
  to attract a renewable energy investor has no institutional-grade analytical
  tool to prepare a credible, data-backed proposal.

• A traditional advisory engagement costs $25,000–$200,000 per project, takes
  4–6 weeks, and is only available from capital-city firms.

• Regional leaders are left with PowerPoint decks, anecdotal evidence, and
  optimistic projections that sophisticated investors see through immediately.

• There is no system — anywhere in the world — that gives a regional government
  the ability to:
  (a) Structure a strategic case with validated data
  (b) Stress-test it with adversarial reasoning
  (c) Score it with quantified, explainable formulas
  (d) Generate institutional-grade documents ready for investor submission
  (e) Do all of this in minutes, not months

BWGA Intelligence AI was built to close this gap.

It was not conceived in a boardroom. It was born from 12 months of firsthand
experience watching dedicated regional leaders struggle to be seen. They had
vision, drive, and genuine opportunity — but lacked the resources to translate
their potential into the language that global investors understand.

This system is the result of that experience.

• A strategic intelligence workflow — not a single AI chat prompt

defensibility is not "AI magic" — it is the repeatable, auditable process
control around output generation. Every number has a formula. Every formula
has a source file. Every recommendation has a vote log.

                    SECTION 6 — THE FORMULA SUITE
                    27+ Proprietary Scoring Engines

================================================================================

Every score the system produces is computed by a deterministic mathematical
formula. Given the same inputs, the same formula always returns the same
output. There is no randomness, no language-model generation in the scoring
pipeline.

THE 5 PRIMARY ENGINES:
----------------------

  1. SPI™  — Success Probability Index
     Purpose: Estimate likelihood of success using 7 weighted factors
     Formula: SPI = Σ(wᵢ × Sᵢ) × P_interaction
     Output:  0–100 score, grade (A/B/C/D), drivers, pressure points
     Source:  services/engine.ts → calculateSPI()

  2. RROI™ — Regional Return on Investment
     Purpose: Project risk-adjusted ROI for a specific region
     Formula: RROI = market×0.3 + regulatory×0.25 + infrastructure×0.25
              + talent×0.2
     Output:  0–100 score, component breakdown
     Source:  services/ReportOrchestrator.ts → RROI calculation

  3. SEAM™ — Stakeholder & Entity Alignment Model
     Purpose: Model the entire ecosystem — stakeholders, incentives,
              conflicts, influence networks
     Output:  Alignment vs conflict signals, influence pressures
     Source:  services/engine.ts → SEAM computation

  4. IVAS™ — Investment Velocity & Activation Speed
     Purpose: Estimate time-to-activation with scenario ranges
     Output:  P10/P50/P90 timelines (months), activation risk flags
     Dependencies: RROI + SPI (computed via DAG)
     Source:  services/algorithms/DAGScheduler.ts → IVAS node

  5. SCF™  — Strategic Cash Flow / Confidence Framework
     Purpose: Unify readiness + capture + timeline + debate consensus
     Formula: SCF = SEAM×0.22 + IVAS×0.22 + SPI×0.22 + RROI×0.22
              + sustainability×0.12
     Output:  Impact P10/P50/P90 (dollar values), viability classification
     Source:  services/engine.ts → computeSCF()

THE 16 DERIVATIVE INDICES:
--------------------------

  Strategic Indices:
    BARNA — Barriers Analysis (regulatory, competitive, capital, cultural)
    NVI   — Network Value Index (partner network connectivity)
    CRI   — Country Risk Index (political, economic, operational)
    FRS   — Flywheel Readiness Score (compounding growth potential)

  Operational Indices:
    CAP   — Capability Assessment Profile (org readiness)
    AGI   — Activation Gradient Index (activation velocity)
    VCI   — Value Creation Index (synergy potential)
    ATI   — Asset Transfer Index (IP/operations transfer complexity)
    ESI   — Ecosystem Strength Index (suppliers, talent, infrastructure)
    ISI   — Integration Speed Index (post-deal integration speed)
    OSI   — Operational Synergy Index (combination synergies)
    TCO   — Total Cost of Ownership (lifecycle costs including hidden)

  Risk Indices:
    PRI   — Political Risk Index (regime stability, expropriation)
    RNI   — Regulatory Navigation Index (compliance complexity)
    SRA   — Strategic Risk Assessment (market/execution/timing)
    IDV   — Investment Default Variance (projection fragility)

ADVANCED EXTENSIONS:
--------------------
    RFI   — Regulatory Friction Index (bottleneck detection)
    RDBI  — Regional Default Bias Index (home-market bias detection)
    AFC   — Activation Friction Coefficient (drag quantification)
    LAI   — Location Attractiveness Index
    Additional Monte Carlo simulation engines (10,000+ iterations)

FORMULA EXECUTION ORDER (DAG — Directed Acyclic Graph):
-------------------------------------------------------
  SPI → RROI → SEAM → IVAS (depends: RROI, SPI) → SCF (depends: all)

  Source: services/algorithms/DAGScheduler.ts

     Purpose: Model the entire ecosystem — stakeholders, incentives,
              conflicts, influence networks
     Output:  Alignment vs conflict signals, influence pressures
     Source:  services/engine.ts → SEAM computation

  4. IVAS™ — Investment Velocity & Activation Speed
     Purpose: Estimate time-to-activation with scenario ranges
     Output:  P10/P50/P90 timelines (months), activation risk flags
     Dependencies: RROI + SPI (computed via DAG)
     Source:  services/algorithms/DAGScheduler.ts → IVAS node

     Purpose: Unify readiness + capture + timeline + debate consensus
     Formula: SCF = SEAM×0.22 + IVAS×0.22 + SPI×0.22 + RROI×0.22

     Output:  Impact P10/P50/P90 (dollar values), viability classification
     Source:  services/engine.ts → computeSCF()
